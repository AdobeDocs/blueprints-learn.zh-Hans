# 区段匹配

区段匹配允许合作伙伴品牌在其各自的Experience Platform环境中共享受众。 品牌的关键是，根据从客户与消费者的直接关系中收集的数据，与客户建立联系。 通过更好的管理、权限和首选项管理系统，营销人员可以与关键合作伙伴进一步增强其经过身份验证的第一方受众。

[!UICONTROL 区段匹配] 是一项允许Experience Platform(AEP)客户(称为 _合作伙伴_)以安全、受管理且对隐私友好的方式，基于通用行业标识符交换区段数据。

该服务使客户能够以安全、中立的方式安全地识别匹配的ID，而无需披露其整个数据库。 合作伙伴仅会为重叠的ID收到指定的属性（区段名称），从而能够以可控的、由同意管理的方式更快、更轻松地共享。

[!UICONTROL 区段匹配] 使用AEP数据管理和同意框架作为其支柱。 它可供所有B2C和B2P Real-time Customer Data Platform客户使用。 的主要功能 [!UICONTROL [!UICONTROL 区段匹配]] 包括：

* 重叠同意客户的分部共享
* 预共享重叠报表，以分析预计匹配量
* 完全集成的DULE策略和权限执行
* 数据共享同意框架骨干
* 用于组织区段和合作伙伴的数据馈送

## 应用程序

品牌到发布者：

弃用第三方Cookie和移动设备广告ID数据对“发布者用例”的影响最大。 此用例对注重以商业模式销售广告的媒体和娱乐行业具有重大影响。 [!UICONTROL 区段匹配] 对于拥有大量第一方受众且希望直接与其广告商协作的出版商而言，这是一条途径。 广告商可以直接与发布者合作，针对发布者属性中匹配的受众发布广告，以进行细粒度定位或潜在促销活动。

### 品牌到品牌

消费者历程从来不是线性的。 例如，客户可能忠于航空公司及其信用卡公司。 使用 [!UICONTROL 区段匹配]，航空公司和信用卡公司可以创建数据合作伙伴关系以了解重叠的受众，然后定制优惠以个性化体验，以提供给每家公司的忠实消费者。

### BU到BU

全球跨国公司在独立运营的业务部门之间开展数据协作方面面临挑战。 由于跨BU的隐私政策、获取或管理权限的不同，可能无法将数据合并到单个沙盒中。

[!UICONTROL 区段匹配] 帮助大型组织中分散的营销团队更有效地协作，同时继续独立运营

## 架构

![区段匹配架构](assets/architecture-segment-match.png)

[!UICONTROL 区段匹配] 不是可以购买数据的数据市场。 它而是一项AEP功能，可与选定的合作伙伴一起处理第一方数据，从而使用隐私和同意控制来帮助进行协作。 [!UICONTROL 区段匹配] 有助于将精力集中在改善客户关系和提升品牌上。 如果存在预先存在的品牌或合作伙伴关系，则此项活动将非常有益。 [!UICONTROL 区段匹配] 体验易于管理、可扩展，并且允许管理员以选择加入和可控的方式共享区段。

[!UICONTROL 区段匹配] 启用：

* 使用标准的人员级别标识符（如经过哈希处理的电子邮件或电话号码）在组织之间安全地移植区段成员身份数据
* 受众共享UI和与通知共享的工作流
* 预共享重叠估计数
* 自助式合作伙伴设置
* 在选择标准化的命名空间（经过哈希处理的电子邮件、经过哈希处理的手机、ECID、IDFA、GAID）上重叠
* 数据共享同意强制实施
* 共享受众生命周期管理
* 共享工作流中的DULE实施
* 每日批量更新

[!UICONTROL 区段匹配] 允许创建互联的客户体验。 支持的持久标识符包括经过哈希处理的电子邮件、经过哈希处理的电话号码以及ECID、IDFA和GAID等标识符。 客户可以构建动态消息，以在品牌沙箱之间匹配和移动受众数据，并具有强大的管理能力、透明度和撤销用于广告和营销活动的功能

## 先决条件

先决条件 [!UICONTROL 区段匹配] 为：

* RT-CDP活动许可
* 支持的标准哈希标识符为SHA256哈希电子邮件、哈希电话、ECID、Apple IDFA和GAID
* 隐私框架和同意策略
* 客户之间已签订数据共享协议

## 安全性

### RBAC

的 [!UICONTROL 区段匹配] 合作伙伴的流量由RBAC保护。 只有拥有适当权限的个人才能启动、接受或管理合作伙伴。 这可以在产品配置文件的数据摄取部分中完成。 需要以下权限：

![Audience Share连接](assets/data-ingestion.png)

| 权限 | 描述 |
|---|---|
| **管理受众共享连接** | 此权限允许您完成合作伙伴握手过程，该过程会连接两个IMS组织以启用 [!UICONTROL 区段匹配] 流量。 |
| **管理受众共享** | 此权限允许您创建、编辑和发布信息源(用于 [!UICONTROL 区段匹配])与积极合作伙伴(管理员用户已与 **受众共享连接** 访问)。 |

请参阅 [官方文档](https://experienceleague.adobe.com/docs/experience-platform/segmentation/ui/segment-match/overview.html?lang=en#understanding-segment-match-permissions) 以了解有关权限的更多信息。

### 连接ID

合作伙伴连接过程由 **[!UICONTROL 连接ID],** 随机生成的标识符，可映射到特定AEP沙箱。 启动和管理合作伙伴沙箱需要此连接ID。 此外，还可以根据需要重新生成连接ID以重新配置合作伙伴连接。

### 治理

具有的任何数据集或数据属性 *C11* 合同标签限制 [!UICONTROL 区段匹配] 服务。 使用这些属性的区段不能用于 [!UICONTROL 区段匹配]. 这提供了可用于或不能用于哪些区段的控制 [!UICONTROL 区段匹配]. 此外，还强制执行创建的自定义策略和营销操作。 默认情况下，策略处于禁用状态，需要启用才能执行。 在共享区段时选择的电子邮件营销和现场广告等限制也会传播并与合作伙伴共享。

### 同意

的同意设置 [!UICONTROL 区段匹配] 可通过以下方式进行管理：

* 在组织级别，在载入过程中使用同意检查的选择退出或选择加入设置。

   此设置确定是否可共享用户数据。 默认设置为选择退出，这表示在AEP客户已经拥有用于数据共享所需的同意协议的情况下，可以共享用户数据。 可以联系Adobe客户经理，并加入额外检查以强制AEP客户明确跟踪同意，从而将此设置更改为选择启用。

* 使用 [同意和首选项字段组](https://experienceleague.adobe.com/docs/experience-platform/xdm/field-groups/profile/consents.html?lang=en).

   此字段组提供单个对象类型字段（同意）以捕获同意和首选项信息。 [!UICONTROL 区段匹配]默认情况下，将包含所有尚未明确退出的标识，例如：

   ```
   "share": {
   `                `"val": "n"
   `     `}
   ```

   通过联系Adobe客户经理，可以更改此设置，以仅包含具有明确选择加入的身份，例如：

   ```
   "share": {
   `                `"val": "y"
   `     `}
   ```

### 警报

在启动合作伙伴连接或与合作伙伴共享区段馈送时，会生成警报。

## 设置工作流

如上所述，使用RBAC管理用于设置合作伙伴连接的工作流。 如果拥有适当的权限，则与合作伙伴沙盒的连接要求共享该沙盒/实例的连接ID。

在从发送合作伙伴请求连接后，必须在接收方批准该连接，以确保安全的合作伙伴设置。 合作伙伴连接握手可确保两个组织之间存在协议，并允许Adobe为 [!UICONTROL 区段匹配] 代表组织处理。 如果连接已批准且处于活动状态，则可从任一方启动区段共享过程。

### 区段共享

仅当选定的标识符存在匹配项时，才会与合作伙伴共享区段。 可能存在一对多的合作伙伴关系，这意味着可以与多个合作伙伴共享区段。

要在设置合作伙伴连接后启动区段共享，发送合作伙伴应创建一个信息源。 然后，选择应从中排除区段数据的营销用例或操作以及持久标识符。 然后，可以将相关区段添加到信息源中以进行共享。

作为此区段共享工作流程的一部分，发送合作伙伴可以在移动任何数据之前通过估计的重叠来发现潜在的高价值区段。

整个流程流程是：

![区段共享](assets/segment-sharing.png)

这些重叠估计可提供关键的洞察、合作伙伴发现和数据来推动数据协作协议。 不会跨沙盒移动客户或区段数据以获取这些重叠的估计量度。 任何给定沙箱中由客户选择且经过预哈希处理的适用身份将添加到可能性数据结构中，该数据结构允许Adobe在它们之间执行并集和交集操作。 这些操作有助于 [!UICONTROL 区段匹配] 从两个不同的沙箱中获取由身份构成的两个数据结构的估计交集，而无需比较实际值

身份重叠过程取决于 **每日完整资料导出** 发件人和接收者沙箱中的数据集，用于标识属于共享区段的常用配置文件。 重叠流程的详细流程如下所示：

![身份重叠流程](assets/overlap-process.png)

在从发送合作伙伴完成区段共享后，接收者会收到有关共享区段馈送的通知。 必须在接收器启用此区段馈送，以便配置文件启动区段成员资格数据流。 只有区段成员资格会被摄取到接收者IMS组织重叠的配置文件片段中，并且不会从发送者向接收者传输任何其他身份。

共享区段位于 `AEPSegmentMatch` 部分 **[!UICONTROL 受众]** 选项卡 **[!UICONTROL 区段生成器]** 和可用于在接收器沙盒中构建区段时包含或抑制受众。

每日重叠过程可在发件人和接收者之间保持区段成员资格同步。 接收者可以为收到的区段馈送禁用配置文件，以暂停区段共享过程。

#### 区段退出/登入

在完整的用户档案导出过程中，用户档案区段成员资格下共享区段ID的状态具有相应的值之一 —  _实现_, _退出_&#x200B;或 _现有_ 以反映当前状态。

在每日身份重叠过程中，如果接收者沙盒中存在相应的身份，则共享区段的这些区段成员资格状态将会发送到接收者以进行摄取。

#### 区段撤销

从发件人撤销/删除区段是一个按需流程，其中从接收者处获取具有撤消区段ID的所有用户档案的列表。 区段ID将从这些标识的区段成员资格中删除，并在接收器上引入。 此操作将覆盖现有区段成员资格片段，该片段将删除该区段的成员资格。
