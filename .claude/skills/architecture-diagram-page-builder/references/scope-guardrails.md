# 范围护栏：架构页面与用例模式页面

Blueprint站点将&#x200B;**架构图页面**&#x200B;与&#x200B;**用例模式页面**&#x200B;分开，因为它们满足不同的读者需求。 本文档定义什么内容属于何处以及如何处理跨越边界的内容。

## 核心区别

- **架构图页面**&#x200B;是顶级可视化引用。 他们回答： *“这些系统如何组合？ 集成点在哪里？ 数据流的形状如何？&quot;* 读者来这里是为了了解自己的方向。
- **用例模式页面**&#x200B;是实施指南。 他们回答： *“如何构建此功能？ 涉及哪些功能？ 什么KPI衡量成功？ 我的实施选项是什么？&quot;* 当读者有用例并且需要发送该用例时，他们就会来到这里。

## 属于体系结构页面

| 类别 | 示例 |
| --- | --- |
| 顶层体系结构 | AEP和应用程序、Experience Cloud营销结构、网络中心与边缘拓扑的概述图 |
| 系统数据流 | 实时与批量摄取路径，中心与边缘之间的配置文件同步，查找与激活流程 |
| 集成点 | 其中AEP与AJO、CJA、Target、Campaign、Marketo、Workfront集成；SDK边界；API界面 |
| 部署拓扑 | Web SDK与移动SDK部署、服务器端转发、边缘节点放置 |
| 应用程序架构 | 如何在系统级别对单个应用程序(AJO、CJA、RTCDP)进行内部结构 |
| 用例模式的指针 | 此架构支持带有链接的X、Y、Z模式 — 架构页面&#x200B;**不**&#x200B;复制该内容 |

## 不属于架构页面

如果您发现自己正在编写以下任何内容，请重定向到用例模式页面（使用`use-case-pattern-builder`技能）：

| 类别 | 为什么它属于别处 |
| --- | --- |
| KPI和度量公式 | 用例模式衡量结果；架构页面不会 |
| 业务目标、业务影响 | KBO内容位于`/help/blueprints/business-objectives/`下；模式引用它 |
| 战术用例示例 | “购物车放弃提醒”、“个性化主页主页”等 — 这些是模式内容 |
| 函数链(`A > B > C > D`) | 功能链构造是用例模式模板的一部分 |
| 人物叙事 | “营销人员玛利亚……” 样式方案属于模式，而不是架构引用 |
| 实施选项 | 多选项实施指南（最适合、它的工作方式、优点、限制）是一种模式结构 |
| 基础/支持函数表 | 这些是模式页部分 |
| 每个用例的先决条件检查列表 | 模式跟踪这些模式；架构页面改为链接到模式 |

## 触发要监视的短语

如果用户描述新页面时提供以下任一短语，请暂停并重新检查范围：

- &quot;KPIs&quot;
- “业务影响”/“业务成果”
- “战术用例”/“示例场景”
- “功能链”
- &quot;implementation options&quot;
- “最适合”
- “优势和限制”
- &quot;前提条件&quot;
- “角色”/“利益相关者”
- “测量”

这些标记不会自动取消页面的资格，但它们表示用户可能需要用例模式页面，而不是架构页面。 在生成之前确认意图。

## 内容漂移时要执行的操作

1. **识别漂移。** 指向跨越边界的特定部分或项目符号。
2. **为用户提供两个选项：**
   - 修剪架构页面中的部分（最常见 — 保持架构页面焦点）。
   - 停止并切换到该内容的`use-case-pattern-builder`（当用户实际需要模式页面时）。
3. **等待确认。** 请勿静默重写或丢弃内容。
4. **如果保留仅体系结构内容**，则在`## Use case patterns supported`下将深层内容替换为链接到相关模式（现有或待创建）的单个项目符号。

## Edge案例

- **页面为半架构，半模式。** 拆分为两个页面 — 一个架构页面（此技能），一个用例模式页面（`use-case-pattern-builder`技能）。 交叉链接它们。
- **体系结构页面描述了一个端到端用例。** 这是一个用例模式，而不是架构页面。 重定向到`use-case-pattern-builder`。
- **架构页面需要显示特定方案的示例数据流。** 如果场景仅为说明性且页面主体位于系统架构级别，则可接受。 将示例保留到一段落，并链接到相关模式以获取完整详细信息。

## 快速测试

在生成之前，询问： *“如果读者登陆此页面，并且希望获得顶级架构引用，他们会获得一个，还是获得一个完成了一半的用例演练？”* 如果是后者，则该页面属于`use-case-pattern-builder`。
