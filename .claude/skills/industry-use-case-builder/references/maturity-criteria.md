# 用例成熟度评估标准

## 成熟度级别

### 基础

**徽章：** `[!BADGE Foundational]{type=Neutral}`

用例是&#x200B;**基础**，当它：

- 映射到单步或事件触发的模式（事件触发的消息传递、批量出站消息激活）
- 具有直接的数据要求（单一事件类型、标准配置文件属性）
- 需要很少或没有AI/ML功能
- 使用标准渠道交付（电子邮件、短信、推送），而无需复杂的编排
- 具有成熟的实施模式和明确的最佳实践
- 需要较少的系统集成（通常为1-2个数据源）

**典型模式：**&#x200B;事件触发的消息传递，批量出站消息激活

**示例：**&#x200B;放弃购物车恢复、约会提醒、服务通知、价格下降提醒、帐单付款提醒

### 新兴

**徽章：** `[!BADGE Emerging]{type=Informative}`

用例是&#x200B;**Emerging**，当它：

- 映射到多步或个性化模式（多步编排历程、已知访客Personalization、行为推荐、匿名访客Personalization）
- 需要行为数据收集和分析
- 使用推荐模型或已知访客配置文件分辨率
- 涉及具有分支逻辑的多接触培养序列
- 具有中等集成复杂性（2-4个数据源）
- 可能需要受众激活或基于区段的定位

**典型模式：**&#x200B;多步骤编排历程、已知访客Web/应用程序Personalization、行为推荐、匿名访客Web Personalization、B2B Audience Activation

**示例：**&#x200B;欢迎系列、药物遵守活动、个性化帐户仪表板、潜在客户评分、内容推荐

### 高级

**徽章：** `[!BADGE Advanced]{type=Caution}`

用例是&#x200B;**高级**，当它：

- 映射到决策或跨渠道编排模式（使用Decisioning的跨渠道历程、Offer Decisioning）
- 需要AI支持的预测或倾向评分
- 同时跨多个渠道使用集中式决策逻辑
- 涉及在多个历程点进行实时决策的复杂编排
- 集成复杂度高（数据源超过4个，实时数据馈送）
- 需要完善的业务规则、优先级管理和频率治理

**典型模式：**&#x200B;使用Decisioning的跨渠道历程，Offer Decisioning

**示例：**&#x200B;通过AI评分防止流失、生命周期阶段产品推荐、交叉销售优化、忠诚度计划个性化、VIP独家优惠

## 评估工作流程

1. 确定用例映射到哪个用例模式
2. 检查上面的“典型模式”列表以快速匹配
3. 如果模式未明确指示成熟度，请评估数据复杂性、AI/ML要求和集成范围
4. 通过推理向用户呈现评估：“基于模式映射({pattern})和{key factors}，我会将此分类为{level}，因为{reasoning}。”
5. 让用户在继续之前确认或覆盖
